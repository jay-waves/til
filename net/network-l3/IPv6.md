## IPv6

ipv6 和 ipv4 是不兼容的，由于侵犯了 ipv4 提供商 (国内 ISP 和云服务商这类地主) 的利益，导致从 ipv4 向 ipv6 过渡困难。

主要的迁移难点如下：
* 报文头完全不同
* ARP 协议被 NDP 协议取代
* IPv6 不再需要 NAT，但是 IPv4 + NAT 的模式已经成熟和普及
* 基于上一点，IPv6 安全管控成本高，内网主机容易获得全球单播地址。防火墙难以识别。

#### ipv6 addr

全球单播地址： 如 `2001:0000:0000:0000:0011:0000:0000:0010/64`，简写为 `2001::11:0:0:10/64` （注意全零字段可以压缩，但只能压缩一次）

其他地址类型：

| 类型 | 子类型                     | 地址                    |
| ---- | -------------------------- | ----------------------- |
| 多播 |                            | `ff00::/8`              |
| 单播 | 未指定地址                 | `::/128`                |
| 单播 | 本地回环地址               | `::1/128`               |
| 单播 | 可聚合全球单播地址         | `2001::/16` `2002::/16` |
| 单播 | IPv4 相容地址              | `0:0:0:0::/96`          |
| 单播 | 链路本地地址（Link-Local） | `fe80::/10`             |
| 单播 | 唯一本地地址（ULA）        | `fd00::/8` `fc00::/7`                         |

Link-local address: 仅用于同一网段通信，由网卡自动生成，不能跨路由器。

ULA（Unique Local Address）：类似局域网内本地地址


#### ipv6 header 

IPv6 报文头固定 40B 长度。

```
+--------+---------------+-----------------------------------------+
|Version | Traffic Class |               Flow Label                |
|  4 bit |    8 bit      |                20 bit                   |
+--------+---------------+-----------------------------------------+
|         Payload Length         |  Next Header  |   Hop Limit     |
|             16 bit             |    8 bit      |     8 bit       |
+------------------------------------------------------------------+
|                    Source Address (128 bit)                      |
|                                                                  |
|                                                                  |
|                                                                  |
+------------------------------------------------------------------+
|                  Destination Address (128 bit)                   |
|                                                                  |
|                                                                  |
|                                                                  |
+------------------------------------------------------------------+
```

* FLow label ：流标识
* Payload lengt ：不包含首部的数据长度
* Next Header：上层协议类型，如 TCP、UDP、ICMPv6 
* Hop Limit：类似 TTL
